These two sets of lists demo what I believe is a bug that I'm experiencing in Firefox 57.0.3 (64-bit) and Firefox Developer Edition 58.0b13 (64-bit) on MacOS Sierra 10.13.2. The orange bars on the Helvetica examples should line up perfectly with the background 22px grid. So, too, on the Verdana example with no numbers next to the list items. The bug is with the numbered Verdana example, which drifts off of the grid. I have also seen this behavior with a web-delivered typeface (Camphor, via Fonts.com). Neither Chrome nor Safari exhibit any difference, typeface to typeface, which seems like it should be the expected behavior.
